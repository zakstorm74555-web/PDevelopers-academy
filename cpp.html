2. Memory Management

In C++, you are the manager. You decide where data goes: Stack (Automatic, fast) or Heap (Manual, huge, dangerous).

int a = 10;           // STACK: Deleted automatically
int* p = new int(20); // HEAP: You must delete this manually!
delete p;             // If you forget this, you get a MEMORY LEAK.
3. The Power of Pointers (*)

A pointer is a variable that stores the Memory Address of another variable. One wrong move here, and you get a Segmentation Fault (App Crash).

int var = 100;
int* ptr = &var; // ptr now holds the physical address of var

cout << ptr;  // Prints something like 0x7ffe... (The Address)
cout << *ptr; // Prints 100 (The Value - Dereferencing)
4. Destructors: Cleaning Up

In other languages, "Garbage Collectors" clean up. In C++, we use Destructors to manually free resources when an object dies.

class PDeveloper {
public:
    PDeveloper() { cout << "Born to Code"; }
    ~PDeveloper() { cout << "Memory Freed"; } // Destructor
};
