2. Deep Dive: Mutable vs Immutable

Understanding memory is crucial. If you don't know the difference between a List and a Tuple, your memory management will fail.

  • Lists (Mutable): Can be changed after creation. [1, 2, 3]
  • Tuples (Immutable): Faster, safer, cannot be changed. (1, 2, 3)
  • Dictionaries: Hash-map based key-value pairs. {"id": 1}
Pro Tip: Use Tuples for data that shouldn't change to save RAM and increase speed.
3. List Comprehensions

The "Pythonic" way to create lists in a single line. It's faster and more readable.

# Normal Way
squares = []
for x in range(10):
    squares.append(x*x)

# Pythonic Way
squares = [x*x for x in range(10) if x % 2 == 0]
4. Decorators (@)

Decorators allow you to modify the behavior of a function without changing its source code. This is "Meta-Programming".

def my_decorator(func):
    def wrapper():
        print("Something before the function.")
        func()
        print("Something after.")
    return wrapper

@my_decorator
def say_hello():
    print("Hello PDevelopers!")
5. Magic/Dunder Methods

These are methods with double underscores __ that let you use Python's built-in syntax with your own objects.

class Book:
    def __init__(self, title):
        self.title = title

    def __str__(self):
        return f"Book Title: {self.title}"

b = Book("Python Pro Guide")
print(b) # This calls __str__ automatically
