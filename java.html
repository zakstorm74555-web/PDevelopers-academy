Java Architecture

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible.

1. The Ecosystem (JDK vs JRE vs JVM)

To run Java, you need to understand how it works under the hood.

  • JVM (Java Virtual Machine): The engine that runs the bytecode.
  • JRE (Java Runtime Environment): JVM + Libraries to run apps.
  • JDK (Java Development Kit): JRE + Tools to write/compile code.
Remember: "Write Once, Run Anywhere" (WORA) is Java's core philosophy.
2. Data Types & Variables

Java is Strongly Typed, meaning every variable must have a declared type.

// Primitive Types
int age = 25;              // Whole numbers
double price = 99.99;      // Decimal numbers
char grade = 'A';          // Single character
boolean isCoder = true;    // True or False

// Non-Primitive Types
String name = "PDevelopers"; // Sequence of characters
3. Control Flow (Loops)

Loops allow you to execute code multiple times efficiently.

// For Loop: When you know how many times to run
for (int i = 0; i < 5; i++) {
    System.out.println("Coding in Jhelum...");
}

// While Loop: Runs as long as condition is true
while (isReady) {
    // Keep working
}
4. Object-Oriented Programming (OOP)

This is the heart of Java. Everything is an Object created from a Class.

// Blueprint (Class)
class Student {
    String name;
    void study() {
        System.out.println(name + " is learning Java.");
    }
}

// Execution (Object)
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();
        s1.name = "Ali";
        s1.study(); // Output: Ali is learning Java.
    }
}
